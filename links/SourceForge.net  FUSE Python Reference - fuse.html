FUSE Python in a Nutshell

To implement a FUSE filesystem in Python, you have to subclass the fuse.Fuse class and implemented a number of methods, most of them similar to those of the C API. This class will be referred to as the filesystem class. If you want to support reading and writing of files, you can either define the required methods as part of the filesystem class, or define a separate class for file operations, which will be called the file class.

To mount the filesystem and handle requests, you have to instantiate the filesystem class and call the main() method.

General Filesystem Methods

The following methods can (or have to) be defined by the filesystem class. Whenever an error occurs, the method should raise an exception of type IOError or OSError and set the errno attribute appropriately. FUSE Python will then return this error code to the application that initiated the syscall.

  • getattr(path)
Defining this method is mandatory for a working filesystem. It

implements the lstat() syscall (stat() calls are automatically resolved to lstat() calls by FUSE). This method should return the attributes of the object located at path.

getattr() Should return an object with the following attributes:
  • st_ino: Inode number. Ignored, unless the use_ino mount option is active.
  • st_dev, st_blksize: These fields are ignored by fuse, but they have to be set to some value (not None).
  • st_mode: Object type and permissions. Should be constructed from the constants defined in the stat package and described in the stat(2) man page.
  • st_nlink: Number of hard links to this file
  • st_uid: User ID of the file owner
  • st_gid: Group ID of the file owner
  • st_rdev: Device ID (if applicable)
  • st_size: Size
  • st_blocks: Number of blocks that are allocated for the file
  • st_atime: Time when the object was last accessed (unix timestamp in seconds since 1970)
  • st_mtime: Time when the object was last modified (unix timestamp in seconds since 1970)
  • st_ctime: Time when the object's metadata was last modified (unix timestamp in seconds since 1970)
If path does not exist, getattr() should return -errno.ENOENT (note the minus).
  • readlink(path)
path is a symbolic link. The readlink() method should return the target of this link.
  • mknod(path, mode, rdev)
Should create the filesystem object path. Type and permissions are given by mode. If path is a device node, then rdev contains the device id, otherwise it is None. Only directories are created with the mkdir() method instead.
  • mkdir(path, mode)
Should create the directory path with permissions mode. Note that mode contains only the permissions and not the type (directory). If you want to add the directory type to the permissions, you have to use mode || stat.S_IFDIR.
  • unlink(path)
Should remove the filesystem object at path. It may have any type except for directory.
  • symlink(target, name)
Should create a symbolic link at name with target target.
  • rename(old, new)
Should rename the filesystem object old to new. Note that new may not be in the same directory.
  • link(target, name)
Should create the filesystem object name as a hardlink to target.
  • fsinit(self)
This method is called when Fuse.main() has been invoked and the filesystem is ready to server requests. You can use it to e.g. start some background threads.

File Class Methods

The following methods form the API of file class. As for the filesystem class, in case of an error the methods should raise an exception of type IOError or OSError and set the errno attribute appropriately.

In order to activate the file class, the filesystem instance has to set its file_class attribute to the file class to use, e.g. in the constructor: 

class fs(fuse):
   def __init__(self):
       fuse.__init__()

       self.file_class = my_file_class
# ...
class my_file_class(object):

   def read(self, len, offset):
   # ...
  • __init__(self, path, flags, mode=None)
If you want the file class constructor to receive the instance of the filesystem class, you can use the following trick: 
   class wrapped_file_class (my_file_class):
           def __iniẗ̈̈__(self2, *a, **kw):
               my_file_class__init__(self2, self, *a, **kw)

Now the my_file_class constructor will receive the filesystem instance as its first argument, followed by path, flags and mode.


File Operation Methods

Instead of using a separate file class, the filesystem class can define the following methods in order to support file operations. They are only used if the filesystem instance does not have a file_class attribute. Whenever an error occurs, the methods should raise an exception of type IOError or OSError and set the errno attribute appropriately. FUSE Python will then return this error code to the application that initiated the syscall.

  • open(path, flags)
  • create(path, flags, mode)
  • read(path, length, offset, fh=None)
Must return a str value. Otherwise you get an 'Invalid argument' error.
  • write(path, buf, offset, fh=None)
  • fgetattr(path, fh=None)
  • ftruncate(path, len, fh=None)
  • flush(path, fh=None)
  • release(path, fh=None)
  • fsync(path, fdatasync, fh=None)

Methods provided by FUSE Python

The following methods are provided by FUSE Python. You can use them when defining your own methods.

  • GetContext()
This method is defined in the filesystem class and returns a dictionary with the following entries:
  • uid: User ID of the process that initiated the syscall that is currently being handled
  • gid: Group ID of the process that initiated the syscall that is currently being handled

Using Threads

Any thread that is started before the Fuse.main() method is invoced will be suspended while main() is running. If need threads running while you are serving requests, you should create them in the filesystem's fsinit() method.
